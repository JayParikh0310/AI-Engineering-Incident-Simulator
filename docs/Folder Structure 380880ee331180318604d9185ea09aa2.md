# High-Level Repository

```
ai-engineering-incident-simulator/

├── frontend/
├── backend/
├── incidents/
├── docs/
├── docker/
├── docker-compose.yml
```

---

# Why Separate Frontend & Backend?

Because:

```
React
     ≠
FastAPI
```

Different deployment lifecycle.

Different dependencies.

Cleaner.

---

# Incidents Folder

This is one of the most important folders.

```
incidents/

├── fastapi-001/
├── fastapi-002/
├── fastapi-003/
```

Example:

```
fastapi-001/

├── public.json
├── private.json

├── broken/
│   ├── main.py
│   ├── routes.py
│   └── db.py

└── golden/
    ├── main.py
    ├── routes.py
    └── db.py
```

Exactly as we designed.

---

# Backend Structure

This is where most projects become messy.

I'd structure it like this:

```
backend/

├── src/

│   ├── api/
│   ├── core/
│   ├── models/
│   ├── schemas/
│   ├── services/
│   ├── repositories/
│   ├── db/
│   └── main.py

├── tests/

├── requirements.txt

└── Dockerfile
```

# Repositories

I highly recommend this.

Many beginners skip it.

Huge mistake.

```
repositories/

├── user_repository.py
├── attempt_repository.py
├── incident_repository.py
```

Purpose:

```
Database access only
```

Example:

```
get_user()

create_attempt()

update_skills()
```

---

# Services

The heart of the backend.

```
services/

├── incident_service.py
├── evaluation_service.py
├── skill_service.py
├── report_service.py
├── auth_service.py
```

Business logic lives here.

---

## incident_service.py

Responsible for:

```
Load incidents

Load public metadata

Load private metadata

Load broken files

Load golden files
```

---

## evaluation_service.py

Responsible for:

```
Build prompts

Call LLM

Validate JSON

Return evaluation
```

---

## skill_service.py

Responsible for:

```
Update mastery

Compute skill progress
```

---

## report_service.py

Responsible for:

```
Generate incident report
```

No LLM.

Pure backend.

---
