# V1 Architecture

# My Recommended V1 Architecture

```
Frontend (React)
       │
       ▼

FastAPI Backend
       │
       ├──────────────► PostgreSQL
       │
       ├──────────────► Incident Engine
       │                    │
       │                    ▼
       │             Incident Files
       │
       └──────────────► AI Evaluator
                        LLM Provider
												    │
												    ▼
												OpenRouter
												    │
												    ▼
												Qwen3 32B
```

This is intentionally simple.

---

# Component 1: Frontend

Responsibilities:

```
Authentication

Dashboard

Incident View

Monaco Editor

Hint Requests

Submission

Reports
```

Nothing else.

No business logic.

No skill calculations.

No evaluation.

---

# Component 2: FastAPI Backend

This becomes the brain.

Responsibilities:

```
Authentication

Load incidents

Store attempts

Store skills

Progress tracking

Generate reports

Call evaluator
```

---

# Component 3: Incident Engine

This is a service inside the backend.

Not a separate microservice.

Something like:

```
backend/

services/

    incident_service.py
```

Responsibilities:

```
Load public.json

Load broken files

Load hints

Load private metadata

Load golden files
```

Think of it as:

```
Filesystem
      ↓
Python Objects
```

---

# Example

Frontend asks:

```
GET /api/v1/incidents/current
```

Backend:

```
Incident Service
      ↓
Load incident
      ↓
Return DTO
```

---

# Component 4: AI Evaluator

Also not a microservice.

Just another service.

```
services/

    evaluation_service.py
```

Responsibilities:

```
Build prompt

Send to LLM

Validate JSON

Return structured response
```

---

# Why Not LangGraph?

This is the interesting question.

Right now we have:

```
Input
   ↓
Prompt
   ↓
LLM
   ↓
JSON
```

That is not a workflow.

That's a single evaluation step.

For V1:

```
evaluation_service.py
```

is enough.

---

# My Recommendation

Don't use LangGraph in V1.

Seriously.

You'll add complexity with almost no benefit.

---

# When LangGraph Becomes Useful

Later:

```
Submission
      ↓

Check syntax

      ↓

Evaluate fix

      ↓

Generate mentor notes

      ↓

Recommend next incident
```

Now there are multiple steps.

Now LangGraph makes sense.

But not yet.

---

# PostgreSQL

Dockerized from day one.

Architecture:

```
FastAPI Container
        │
        ▼
Postgres Container
```

Using Docker Compose.

---

# Future Docker Sandbox

This is important.

I want to design V1 so V4 doesn't require rewrites.

Future architecture:

```
Frontend
    │
    ▼

Backend
    │
    ├── Incident Engine
    │
    ├── Evaluator
    │
    └── Sandbox Manager
             │
             ▼
       Docker Containers
```

Notice:

We're reserving a place for it.

But not building it.
