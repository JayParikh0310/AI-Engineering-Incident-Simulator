# API Design

I want our APIs to follow a simple rule:

```
Frontend should never know:
- Incident storage format
- Database schema
- Skill calculations
- Evaluation logic
```

Frontend asks for information.

Backend decides everything.

All V1 backend routes use the `/api/v1` prefix.

---

# Authentication APIs

## Register

```
POST /api/v1/auth/register
```

Request:

```
{
  "username":"jay",
  "email":"jay@example.com",
  "password":"..."
}
```

Response:

```
{
  "access_token":"...",
  "token_type":"bearer",
  "user": {
    "id":"...",
    "username":"jay",
    "email":"jay@example.com",
    "created_at":"..."
  }
}
```

---

## Login

```
POST /api/v1/auth/login
```

Request:

```
{
  "username_or_email":"...",
  "password":"..."
}
```

Response:

```
{
  "access_token":"...",
  "token_type":"bearer",
  "user": {
    "id":"...",
    "username":"jay",
    "email":"jay@example.com",
    "created_at":"..."
```

Simple.

# Incident APIs

This is the heart of the application.

---

## Get Current Incident

```
GET /api/v1/incidents/current
```

Response:

```
{
  "id": "fastapi-001",

  "title": "Orders API Startup Failure",

  "difficulty": "easy",

  "scenario": {...},

  "logs": [...],

  "visible_files": [
    "main.py",
    "routes.py",
    "db.py"
  ],

  "files": {
    "main.py": "...",
    "routes.py": "...",
    "db.py": "..."
  }
}
```

Notice:

No hints.

No root cause.

No private data.

---

## Request Hint

```
POST /api/v1/incidents/{incident_id}/hint
```

Request:

```
{}
```

Response:

```
{
  "level":1,

  "text":"The issue occurs before FastAPI starts."
}
```

Backend tracks:

```
Hint 1 used
```

in database.

---

## Submit Attempt

This is the most important endpoint.

```
POST /api/v1/attempts/submit
```

Request:

```
{
  "incident_id":"fastapi-001",

  "user_confidence":80,

  "files": {
    "main.py":"...",
    "routes.py":"...",
    "db.py":"..."
  }
}
```

---

Response:

```
{
  "passed":true,

  "score":90,

  "summary":
"Circular import successfully removed.",

  "feedback": [
"Good fix.",
"No new issues introduced."
  ],

  "incident_completed":true
}
```

# One Improvement I'd Make

Instead of:

```
POST /api/v1/incidents/hint
```

I would actually do:

```
POST /api/v1/incidents/{incident_id}/hint
```

Cleaner REST style.

Similarly:

```
GET /api/v1/incidents/current

POST /api/v1/incidents/{id}/submit
```

would also work.

Not critical, but cleaner.
