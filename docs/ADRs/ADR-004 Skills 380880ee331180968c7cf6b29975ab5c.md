# ADR-004 Skills

## Status

Accepted

## Context

The platform tracks a user's debugging ability across multiple engineering concepts. Future versions will use this information for recommendations, adaptive learning paths, AI mentor feedback, and personalized incident selection.

We need a simple V1 skill system that is easy to understand, easy to implement, and can be extended later.

## Decision

Skills will be stored in the `user_skills` table.

Each skill record contains:

- user_id
- skill_name
- mastery_score
- attempts_on_skill
- successes_on_skill
- last_updated

Incidents define which skills they test using weighted metadata.

Example:

```json
{
  "skills": [
    {
      "name": "imports",
      "weight": 0.8
    },
    {
      "name": "project_structure",
      "weight": 0.2
    }
  ]
}
```

Skill updates are performed by backend rules, not by the LLM.

The LLM identifies demonstrated concepts, and the backend applies deterministic score updates.

## Hint Policy

Using hints does not reduce mastery score.

Using hints only affects the incident score.

Reason:

Using hints is considered part of the learning process.

## Completion Policy

An incident is considered completed when:

- root_cause_fixed == true
- introduced_new_issues == false

Completion updates relevant skills.

## Future Expansion

Future versions may use:

- AI recommendations
- Personalized learning paths
- Mentor analysis
- Weakness targeting

These systems will consume skill data but will not change the storage model.
