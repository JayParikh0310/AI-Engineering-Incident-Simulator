# ADR-003 Evaluation Engine

# 1. What Gets Sent To The LLM?

This is where many AI projects go wrong.

Do NOT send everything.

Do NOT send entire history.

Do NOT send every attempt.

For V1 I'd send:

```
{
  "incident": {
    "title":"...",
    "scenario":"...",
    "logs": [...]
  },

  "root_cause": {
    "type":"circular_import",
    "description":"..."
  },

  "golden_files": {...},

  "broken_files": {...},

  "user_submission": {...},

  "learning_objectives": [...]
}
```

# The Evaluator Output

I would merge everything into one schema.

Something like:

```
{
  "root_cause_fixed":true,

  "introduced_new_issues":false,

  "confidence":0.92,

  "concepts_demonstrated": [
"imports"
  ],

  "concepts_missing": [],

  "summary":"The circular import was correctly removed.",

  "feedback": [
"Good understanding of import dependencies.",
"No new issues introduced."
  ],

  "recommended_skill_updates": {
    "imports":5
  }
}
```

Backend decides:

```
ifroot_cause_fixedandnotintroduced_new_issues:
passed=True
else:
passed=False
```

# 3. How Do Skill Scores Update?

I would keep this surprisingly simple.

Example:

Incident:

```
{
  "skills": [
    {
      "name":"imports",
      "weight":1.0
    }
  ]
}
```

---

Pass:

```
imports +5
```

Pass with no hints:

```
imports +7
```

Pass first attempt:

```
imports +8
```

---

Fail:

```
imports -1
```

or even:

```
imports unchanged
```

I actually prefer:

```
No penalty.
```

# 4. How Should Hints Affect Scoring?

I wouldn't reduce mastery.

I would reduce incident score.

Example:

```
Base Score = 100
```

Hint 1:

```
-5
```

Hint 2:

```
-10
```

Hint 3:

```
-20
```

Final:

```
100
95
85
65
```

depending on hints used.

# 5. When Is An Incident Completed?

I would define:

```
Completed =
Passed Once
```

Simple.

Example:

```
Attempt 1
Fail

Attempt 2
Fail

Attempt 3
Pass
```

Incident becomes:

```
Completed
```

and next incident unlocks.

# What About Reveal Root Cause?

We discussed this earlier.

Example:

```
Attempt 5
Still failing
```

User clicks:

```
Reveal Root Cause
```

Then:

```
Status:
Studied
```

not

```
Completed
```

Important distinction.

---

# 6. How Do We Keep AI Evaluation Consistent?

This is probably the hardest part of V1.

I would NOT ask:

```
Did user solve this?
```

Too vague.

Instead force structured output.

Example:

```
{
  "root_cause_fixed":true,

  "introduced_new_issues":false,

  "feedback":"...",

  "concepts_demonstrated": [...],

  "concepts_missing": [...]
}
```

Use Pydantic validation.

If response is invalid:

```
Retry LLM
```

## Pass / Fail Rules

```
PASS

root_cause_fixed == true

AND

introduced_new_issues == false
```
